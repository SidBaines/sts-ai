# scripts

CLI entry points and build tooling. See the top-level [`CLAUDE.md`](../CLAUDE.md) for project goals and repo-wide rules.

## Contents

- `build_lightspeed.sh` ‚ÄĒ clones `sts_lightspeed`, applies `patches/sts_lightspeed_python_api.patch` (only if not already applied), and builds the `slaythespire` pybind module. Run this before any rollout.
- `run_rollout.py` ‚ÄĒ runs a single hybrid rollout from the CLI. Always invoke with `PYTHONPATH=src`.
- `run_batch.py` ‚ÄĒ batch rollouts over a seed range/list. Seeds come from `--seeds`, `--seed-start/--seed-count`, or `--seeds-config configs/frozen_seeds.json --split {smoke,dev,eval}`. For `--agent mlx` add `--thinking` (captures `<think>` chain-of-thought into `agent.thinking`), `--max-tokens`, `--temperature`. Use `--seed-timeout-seconds` for LLM/thinking runs (per-seed subprocess + kill) ‚ÄĒ thinking mode can hang on the seed-2-class UB and is slow.
- `run_sweep.py` ‚ÄĒ batched multi-model sweep: models √ó {thinking on/off} √ó seeds via the cross-rollout orchestrator (`sts_ai.parallel_rollout`, `--batch-size` K). Writes per-rollout JSONL + `.meta.json` under `<output-dir>/<agent_label>/`. One model load is reused across both thinking modes; a model that fails to load is skipped. This is the throughput path (~K√ó via batched generation); `run_rollout.py`/`run_batch.py` remain the serial path. `--backend {mlx,vllm}` (default `mlx`) selects the inference engine ‚ÄĒ `mlx` for Apple Silicon, `vllm` for NVIDIA/CUDA. **For `--backend vllm`, run ONE model per process** (vLLM does not free GPU memory between in-process loads): pass a single model per invocation and loop in the shell ‚ÄĒ `run_sweep` prints a warning and does only best-effort cleanup if given >1 model. **Measured per-model rollout speeds (for planning how long a run takes) are in [`docs/throughput_benchmarks.md`](../docs/throughput_benchmarks.md).**
- `runpod/` ‚ÄĒ orchestration for running the sweep on rented NVIDIA GPUs (A40s): `setup_pod.sh` (apt deps + venv + `.[vllm]` + build sim), `run_sweep_on_pod.sh` (loops one model per `run_sweep.py` process, both reasoning arms, continue-on-failure), `sync_back.sh` (crash-resilient local rsync loop), per-pod model lists, and a `README.md` runbook (pod creation via the `runpod-spinup` skill, CUDA‚Č•12.1 caveat, two-pod split, teardown). See [`runpod/README.md`](runpod/README.md).
- `compare_models.py` ‚ÄĒ per-model/mode comparison report over a sweep tree (outcomes, behaviour proxies incl. full-block-forgone & lethal-taken from `affordances`, reasoning length/latency from per-decision tokens, invalid-JSON rate).
- `summarize_rollouts.py` ‚ÄĒ JSONL rollouts ‚Üí summary CSV.
- `compute_risk_proxies.py` ‚ÄĒ JSONL rollouts ‚Üí risk-event CSV + aggregate JSON (deterministic, computed from stored traces).
- `visualize_rollout.py` ‚ÄĒ Streamlit viewer for rollout traces (click-through / 1-decision-per-second autoplay). Out-of-combat decisions show deck/relics/potions + chosen action + reasoning + `<think>`; **combat decisions render a board** (enemy panels with HP/block/intent, a player panel with HP/block/energy/powers, and the hand as neutral card tiles with the chosen card + targeted enemy highlighted). Tiles are colour-by-state (chosen / unaffordable), **not by card type** ‚ÄĒ type isn't in the records (the "records only" data scope). Needs the `viz` extra. Run: `PYTHONPATH=src .venv/bin/streamlit run scripts/visualize_rollout.py`. All parsing lives in the unit-tested `sts_ai.rollout_view` (no Streamlit import there); the script is a thin render shell.

## Area gotchas

- **`--max-tokens` must be large for reasoning/thinking models (default 4096).** A small cap (e.g. 256) truncates the model mid-thought, so it never emits the closing JSON: `parse_json_action` fails, the decision is marked invalid, and the loop **silently falls back to action 0** ‚ÄĒ the "policy" degenerates to "always the first legal action" (and every reasoning arm then produces identical, meaningless aggregates). This was rediscovered the hard way in a sweep; the eval's `invalid_rate` + `completion_tokens` (pinned at the cap) make it obvious. Raising the cap is **free for no-thinking models** (generation stops at EOS ~60-90 tokens, well under the cap), so the high default costs nothing there.
- **`run_rollout.py` deletes the output file if it already exists** before writing. Pass a fresh `--output` path if you need to keep a prior trace.
- Records are appended per-decision during the run; a crash mid-rollout leaves a partial JSONL file.
- **Streamlit HTML rendering in `visualize_rollout.py` has two sharp edges.** (1) Markdown treats any line indented ‚Č•4 spaces as a code block, so each HTML tile/panel must be emitted as a **single-line** string (no pretty-printed newlines) via `st.markdown(..., unsafe_allow_html=True)`. (2) A rerun (e.g. autoplay/`st.rerun`) re-renders the page from scratch, so the `<style>` block is **re-emitted every run** rather than once ‚ÄĒ don't guard it behind a session flag or the tiles lose their CSS.
